Seulement deux cibles ici : des fonctions pures isolées testables sans BDD ni réseau. La validation des données est gérée par l'ORM — pas besoin de la retester. Les flux complets (login, création…) sont couverts par l'E2E en staging.
🔐 Hashage mot de passe — bcrypt
Happy path
  • hashPassword(pwd) → hash non vide
  • hash ≠ password original
  • hash commence par $2b$ (signature bcrypt)
  • comparePassword(pwd, hash) → true
Cas d'erreur
  • comparePassword(mauvais, hash) → false
  • hashPassword('') → throw Error
  • hashPassword(null) → throw Error
🔑 Middleware JWT — vérification token
Happy path
  • Token valide → next() appelé
  • req.user contient userId + role
  • Payload correctement extrait
Cas d'erreur → 401
  • Header Authorization absent → 401
  • Token malformé → 401
  • Token expiré → 401
  • Signature invalide → 401
Vitest — hashage + middleware JWT
import { hashPassword, comparePassword } from '../utils/auth'
import { authMiddleware } from '../middlewares/auth'
import jwt from 'jsonwebtoken'

// ─── Hash ───────────────────────────────────────────────
describe('hashPassword', () => {
  it('produit un hash bcrypt valide', async () => {
    const hash = await hashPassword('secret42')
    expect(hash).toMatch(/^\$2b\$/)
    expect(hash).not.toBe('secret42')
  })

  it('compare correctement', async () => {
    const hash = await hashPassword('secret42')
    expect(await comparePassword('secret42', hash)).toBe(true)
    expect(await comparePassword('mauvais',  hash)).toBe(false)
  })

  it('rejette un password vide', async () => {
    await expect(hashPassword('')).rejects.toThrow()
  })
})

// ─── Middleware JWT ──────────────────────────────────────
describe('authMiddleware', () => {
  const mockRes = () => ({ status: vi.fn().mockReturnThis(), json: vi.fn() })
  const next = vi.fn()

  it('token valide → next() + req.user rempli', () => {
    const token = jwt.sign({ userId: 1, role: 'user' }, 'SECRET')
    const req = { headers: { authorization: `Bearer ${token}` } }
    authMiddleware(req, mockRes(), next)
    expect(next).toHaveBeenCalled()
    expect(req.user.userId).toBe(1)
  })

  it('token absent → 401', () => {
    const res = mockRes()
    authMiddleware({ headers: {} }, res, next)
    expect(res.status).toHaveBeenCalledWith(401)
  })

  it('token expiré → 401', () => {
    const token = jwt.sign({ userId: 1 }, 'SECRET', { expiresIn: '-1s' })
    const res = mockRes()
    authMiddleware({ headers: { authorization: `Bearer ${token}` } }, res, next)
    expect(res.status).toHaveBeenCalledWith(401)
  })
})
vi.fn() / vi.mock() toMatch · toBe · rejects.toThrow toHaveBeenCalledWith aucune BDD · aucun réseau · req/res mockés

Happy path
  • GET /users → 200 + liste (admin)
  • GET /users/:id → 200 + user
  • PATCH /users/:id → 200 + modifié
  • DELETE /users/:id → 204 (admin)
Erreurs

  • Non-admin sur GET /users → 403
  • User inexistant → 404
🍸 /cocktails — CRUD cocktails
Happy path
  • GET /cocktails → 200 liste publique (sans token)
  • GET /cocktails/:id → 200 détail public
  • POST /cocktails → 201 (token requis)
  • PUT /cocktails/:id → 200 (propriétaire)
  • DELETE /cocktails/:id → 204 (propriétaire)
Erreurs
  • POST sans token → 401
  • PUT/DELETE sans token → 401
  • Cocktail inexistant → 404
⚖️ Règle d'ownership — priorité de test
AUTORISÉ
  • User A modifie son propre cocktail → 200
  • User A supprime son propre cocktail → 204
  • Admin modifie n'importe quel cocktail → 200
INTERDIT → 403
  • User B modifie le cocktail de A → 403
  • User B supprime le cocktail de A → 403

Jest + Supertest — ownership + codes retour
describe('PUT /cocktails/:id', () => {
  let tokenA, tokenB, cocktailId

  beforeAll(async () => {
    // Seed : 2 users + 1 cocktail appartenant à A
    tokenA     = await loginAs('alice@test.com')
    tokenB     = await loginAs('bob@test.com')
    cocktailId = await seedCocktail(tokenA, { name: 'Mojito' })
  })

  afterAll(() => db.clean())

  it('200 — propriétaire modifie son cocktail', async () => {
    const res = await request(app)
      .put(`/cocktails/${cocktailId}`)
      .set('Authorization', `Bearer ${tokenA}`)
      .send({ name: 'Mojito Royal' })
    expect(res.status).toBe(200)
  })

  it('403 — autre user tente de modifier', async () => {
    const res = await request(app)
      .put(`/cocktails/${cocktailId}`)
      .set('Authorization', `Bearer ${tokenB}`)
      .send({ name: 'Hack' })
    expect(res.status).toBe(403)
  })

  it('401 — sans token', async () => {
    const res = await request(app)
      .put(`/cocktails/${cocktailId}`)
      .send({ name: 'Hack' })
    expect(res.status).toBe(401)
  })
})

Tests de Charge — k6 ⚡
k6 · routes API ciblées · p95 · taux d'erreur
▼
Endpoints ciblés
  • GET /cocktails — lecture publique intensive
  • POST /auth/login — pic de connexions simultanées
  • POST /cocktails — écritures concurrentes (authentifié)
  • GET /cocktails/:id — accès détail sous charge
Seuils à valider
  • p95 temps de réponse < 500ms
  • Taux d'erreur < 1%
  • Aucun 500 sous charge nominale
  • Pool BDD : pas de timeout connexion
  • Mémoire stable dans le temps
k6 — scénario cocktail app
import http from 'k6/http'
import { check, sleep } from 'k6'

export const options = {
  stages: [
    { duration: '30s', target: 30  },  // montée progressive
    { duration: '1m',  target: 100 },  // maintien en charge
    { duration: '20s', target: 0   },  // redescente
  ],
  thresholds: {
    'http_req_duration': ['p(95)<500'],
    'http_req_failed':   ['rate<0.01'],
  }
}

export default function () {
  // Lecture publique
  const list = http.get('https://api.staging/cocktails')
  check(list, { 'status 200': r => r.status === 200 })

  // Détail d'un cocktail
  const detail = http.get('https://api.staging/cocktails/1')
  check(detail, { 'status 200': r => r.status === 200 })

  sleep(1)
}
